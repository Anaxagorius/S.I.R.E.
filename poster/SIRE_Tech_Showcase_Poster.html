🎯 What Is S.I.R.E.?

S.I.R.E. is an immersive, instructor-led emergency preparedness simulator built for teams that need to rehearse high-pressure decisions before a real crisis hits. An instructor selects a scenario — fire, active threat, cyberattack, flood, and more — and trainees join via a unique session code. Events escalate automatically on a timer while the instructor can inject curveballs in real time. Every action is logged, every decision is visible, creating a rich post-exercise debrief record.

Built as a full-stack JavaScript project with React on the front end and Node.js + Socket.IO on the back end, S.I.R.E. delivers < 300 ms perceived update latency across LAN-class networks and supports ≥ 10 concurrent trainees per session.

🧠 Architecture Decision Highlights
Full-Stack JavaScript
Single-language stack (Node.js + React) reduces cognitive overhead and enables shared patterns across front and back end.
Socket.IO for Real-Time
Chosen over raw WebSockets for built-in fallbacks, automatic reconnection, and room-based session isolation.
In-Memory Session Store
Zero-dependency MVP storage that meets the ≥10 trainees/session target without introducing a database deployment burden.
Multi-Cloud Deployment
Dockerfile + Railway / Render / Heroku configs ensure zero-cost academic hosting with no vendor lock-in.
